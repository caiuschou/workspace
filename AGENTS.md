# 文档

- 在用户给出明确要求时，把讨论的方案写入文档
- 考虑主题的大小，有必要的话拆分文档到不同的文件
- 拆分时考虑创建对应的目录
- ideas 目录是用户专用
- 基于 `docs/` 进行开发，任何一个任务，都要规划任务到表格中，完成后标记为完成。

# Rust

- 代码尽可能的按文件组织
- 一个类型一个文件
- 测试文件独立放置
- 每一个类型、方法都需要有详细的注释
- 注释要包括其交互的方法和类型
- 注释使用英文

# Git

- 提交代码的时候，消息使用英文
- 分类提交，每一个类别一次提交
- 不提交 thirdparty 的内容

# 单元测试
- 把单元测试拆分到单独的目录
- BDD 的风格
- 最好单元测试能成为文档
- 单元测试一定要有详细的用例方法的注释
