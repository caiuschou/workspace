
# 通用

- 一切方案都要简单
- 一切编码都要简单
- 任何事情
  - 拆解成若干步骤
  - 可以按照树形思维拆解
  - 写入文档
  - 完成后标记
- 任何内容和思考都要仔细论证
- 如果要做的事情有一定的规模，请使用 work tree
- 做的事情要验证
  - 很小的事情就开始验证
  - 检查内容
  - 如果有代码
  - 检查代码
  - 编写单元测试
  - 执行单元测试
  - 如果单元测试不够，编写集成测试
  - 执行集成测试
- 考虑多次论证
- 一定要上网搜索资料，研究，包括最佳实践
- AI 对话过程使用中文
- 不用读取 archive 的所有内容

# 文档

- 在用户给出明确要求时，把讨论的方案写入文档
- 考虑主题的大小，有必要的话拆分文档到不同的文件
- 拆分时考虑创建对应的目录
- ideas 目录是用户专用
- 基于 `docs/` 进行开发，任何一个任务，都要规划任务到表格中，完成后标记为完成。

# Rust

- 代码尽可能的按文件组织
- 一个类型一个文件
- 测试文件独立放置
- 每一个类型、方法都需要有详细的注释
- 注释要包括其交互的方法和类型
- 注释使用英文

# Git

- 提交代码的时候，消息使用英文
- 分类提交，每一个类别一次提交
- 不提交 thirdparty 的内容
